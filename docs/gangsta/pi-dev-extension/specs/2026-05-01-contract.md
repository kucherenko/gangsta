# Contract: pi.dev Extension for Gangsta Agents Framework

## Objective

Add a pi.dev coding agent extension to the Gangsta Agents Framework repository, following the established multi-platform adapter pattern. The extension integrates Gangsta into the pi.dev runtime by injecting a condensed bootstrap system prompt at agent start, registering three slash commands (`gangsta:heist`, `gangsta:go`, `gangsta:abort`), and providing a platform-specific tool mapping reference file. The adapter ships as project-local files under `.pi/` committed to the master branch. No changes are made to the root `package.json` or to any existing skill files beyond adding one reference document under `skills/using-gangsta/references/`.

## Requirements

### Functional Requirements

1. FR-001 — A TypeScript extension entry point must exist at `.pi/extensions/gangsta/index.ts` that exports a default factory function conforming to the pi.dev `ExtensionAPI` imperative callback interface.

2. FR-002 — The extension must register a `before_agent_start` event handler that injects a condensed Gangsta bootstrap prompt into the agent's system context at session start.

3. FR-003 — The condensed bootstrap prompt injected by FR-002 must include, at minimum: the Borgata hierarchy summary, the 1% skill invocation rule, a skill table listing all skills present in `skills/*/SKILL.md` at the time of implementation with their relative paths, and the intent routing table. The current count at contract-signing date is 21; this count must be updated if skills are added or removed.

4. FR-004 — The extension must register three slash commands via `pi.registerCommand`: `gangsta:heist`, `gangsta:go`, and `gangsta:abort`.

5. FR-005 — Each slash command registered in FR-004 must read its content at invocation time from the corresponding file in the `commands/` directory (`commands/heist.md`, `commands/go.md`, `commands/abort.md`) and deliver that content to the user via `ctx.ui.notify()`. Path resolution must use `node:path.resolve(__dirname, '../../../commands/')` as the canonical strategy; if `__dirname` is unavailable at runtime, the fallback must be `node:path.resolve(process.cwd(), 'commands/')` with a logged warning.

6. FR-006 — A `package.json` must exist at `.pi/extensions/gangsta/package.json` declaring the extension entry point via the `pi.extensions` field and specifying zero runtime dependencies.

7. FR-007 — An installation guide must exist at `.pi/INSTALL.md` documenting how to enable the extension in a pi.dev project, listing all three accepted risks (`__dirname` path resolution under jiti, `ctx.ui.notify()` channel behavior, condensed bootstrap prompt drift), and confirming the zero-dependency constraint.

8. FR-008 — A platform tool mapping reference file must exist at `skills/using-gangsta/references/pi-tools.md` mapping pi.dev platform tool names to the canonical Claude Code tool names used throughout Gangsta skill files, following the structure of the existing `opencode-tools.md` reference.

9. FR-009 — `scripts/validate.sh` must be updated to add `.pi/extensions/gangsta/` to the scan exception list so the identifier validator does not flag extension files as NC-002 violations.

### Non-Functional Requirements

1. NFR-001 — The root `package.json` must remain at zero runtime dependencies. The `.pi/extensions/gangsta/package.json` must declare `"dependencies": {}`.

2. NFR-002 — No file under `.pi/` or `skills/using-gangsta/references/pi-tools.md` may contain Gangsta-internal spec identifiers matching the pattern `\b(FR|NFR|WP)-\d+\b`. This is a binding constraint per NC-002.

3. NFR-003 — The extension must use only Node.js built-in modules (`node:fs`, `node:path`) for all file I/O operations. No third-party npm packages may be imported.

4. NFR-004 — All work must be committed directly to the master branch with no feature branch or worktree, per the Don's explicit instruction.

5. NFR-005 — The extension entry point must not exceed 150 lines of TypeScript. The implementation is intentionally minimal.

## Architectural Decisions

From the Grilling consensus (2 rounds, Don declared consensus after Round 2):

- **Imperative callback API:** The extension factory receives `pi: ExtensionAPI` and calls `pi.on()` and `pi.registerCommand()` imperatively. It does not return a declarative object. This is the verified pi.dev extension API.

- **No `resources_discover` dependency:** This API was confirmed to not exist in pi.dev's `ExtensionAPI`. Skill availability is delivered entirely via the injected bootstrap prompt listing relative paths.

- **Condensed bootstrap prompt:** The full `skills/using-gangsta/SKILL.md` is not injected verbatim. A hand-authored condensed version is written directly into `index.ts`. This is a deliberate trade-off accepted by the Don.

- **Node.js built-ins for path resolution:** `fs.readFileSync` and `path.join(__dirname, ...)` are used for command file reads. The `__dirname` reliability risk under jiti is an accepted risk (Don confirmed "move forward").

- **`ctx.ui.notify()` for command output:** The accepted delivery channel for command content. Whether this injects into LLM context or renders as a UI notification is a known unknown documented in INSTALL.md.

## Grilling Conclusions

### Key Decisions

- TypeScript imperative API confirmed as the correct extension model for pi.dev
- `resources_discover` removed (API does not exist); skills listed in bootstrap prompt instead
- Zero runtime dependencies enforced at `.pi/extensions/gangsta/package.json` level
- `commands/` directory confirmed to exist; command handlers read from existing files
- Condensed bootstrap prompt chosen over full SKILL.md injection to reduce size risk
- Work targets master branch directly per Don's instruction

### Rejected Alternatives

- `resources_discover` skill loading: API does not exist in pi.dev's `ExtensionAPI`
- Runtime skill file reads at agent start: adds I/O at startup with no recovery path
- External npm dependencies: violates zero-dependency constraint
- Separate extension file per command: unnecessary complexity
- Markdown-only context file: rejected by Don in favour of full TypeScript extension

### Unresolved Objections

- `__dirname` unreliable under jiti — Risk: MEDIUM — Mitigation: none; Don accepted
- `ctx.ui.notify()` may be wrong channel for command output — Risk: LOW — Mitigation: documented in INSTALL.md; Don accepted
- Condensed bootstrap prompt drift — Risk: MEDIUM — Mitigation: none; Don accepted

## Acceptance Criteria

1. `ls .pi/extensions/gangsta/` returns `index.ts` and `package.json`.
2. `.pi/extensions/gangsta/package.json` contains a `pi.extensions` array referencing `"index.ts"` and `"dependencies": {}`.
3. `.pi/INSTALL.md` exists and contains installation instructions.
4. `skills/using-gangsta/references/pi-tools.md` exists and contains a tool mapping table.
5. `scripts/validate.sh` contains a scan exception entry for `.pi/extensions/gangsta/`.
6. `grep -r "FR-\|NFR-\|WP-" .pi/ skills/using-gangsta/references/pi-tools.md` returns no matches.
7. `cat package.json | grep '"dependencies"'` at repo root returns no new entries.
8. `git log --oneline master ^origin/master` returns exactly one line (single atomic commit on master).

## Out of Scope

- Changes to `commands/heist.md`, `commands/go.md`, `commands/abort.md` — these files exist and are used as-is.
- Changes to any existing `skills/*/SKILL.md` files other than adding the `pi-tools.md` reference file.
- npm package distribution of the extension.
- Custom pi.dev tools beyond the three slash commands.
- TUI widgets, keyboard shortcuts, or provider registration.
- Global (`~/.pi/`) extension installation instructions.
- Any changes to `.claude-plugin/`, `.cursor-plugin/`, `.opencode/`, or `gemini-extension.json`.

## Open Risks

- `__dirname` resolution under jiti may cause command file reads to fail at runtime — Risk: MEDIUM — No mitigation; Don accepted.
- `ctx.ui.notify()` may not inject content into LLM conversation context — Risk: LOW — Documented in INSTALL.md; Don accepted.
- Condensed bootstrap prompt will drift from `using-gangsta/SKILL.md` as the framework evolves — Risk: MEDIUM — No automated sync; Don accepted.
